# Vestige — Positioning & Voice Reference

## The Core Differentiator

Competitors measure repositories. **Vestige reads them.**

No existing tool reads a diff and asks *"why did someone make this decision?"* None surface the reasoning behind a deleted function, why a dependency was quietly swapped, or what problem a 3am hotfix was actually solving.

**Vestige is a decision archaeology tool.**

---

## The Target User

Not a researcher studying project health.

- A developer who just joined a codebase, staring at a weird architectural choice wondering *"why is this like this?"*
- You, running it on an interviewer's own repo and surfacing something they themselves forgot about.

That's an entirely different job to be done.

---

## Language to Use

| Use | Avoid |
|-----|-------|
| Decision archaeology | Repository analysis |
| Reading the codebase | Measuring the codebase |
| Why this exists | What this does |
| Surfacing reasoning | Generating reports |
| Decisions buried in history | Code metrics |

---

## What to Put Foremost in Copy

1. **The question no other tool asks** — "Why did someone make this decision?"
2. **The user's actual moment of need** — staring at something weird, wondering why
3. **The competition reframe** — they measure, we read
4. **The archaeology metaphor** — digging up decisions, not auditing health
